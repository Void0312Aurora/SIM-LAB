# 第一版资产清单

## 1. 文档目的

本文档用于定义项目第一版应包含的资产范围、优先级、精度目标以及推荐的数据来源或生成方式。

这里的“资产”不只指模型、贴图和角色，也包括支撑模拟运行的结构化数据与语义图层。

本清单基于当前共识：

- 首发版本以第三人称上帝视角、俯视角或 2.5D 视角为主
- 首发目标是“结构可信、模拟可信”，不是近距离写实
- 第一人称沉浸式体验为后续扩展方向

## 2. 精度分层

为了控制范围，第一版资产精度分为三档：

- `高`：影响导航、模拟结果、通行逻辑或战术判断，必须尽量准确
- `中`：影响区域感知、空间语义或局部策略，允许一定近似
- `低`：主要影响观感，不应阻塞第一版原型

从不同维度看，第一版精度目标建议如下：

- 几何精度：道路、建筑轮廓、围墙、水体和大障碍物优先保证
- 拓扑精度：道路连通性、步行可达性、入口可达性优先保证
- 语义精度：建筑大类用途、关键 POI、危险点和安全点优先保证
- 表现精度：立面细节、招牌、窗户、材质微差异可以后置

## 3. 优先级定义

- `P0`：首发必须具备，没有就无法形成完整玩法或模拟闭环
- `P1`：首发强烈建议具备，会显著提升可信度和可玩性
- `P2`：后续增强项，不应阻塞第一版

## 5. 第一版最小闭环资产集合

如果只做一个可跑通的 MVP，建议最少具备以下资产：

- 道路中心线、路口节点、人行区域
- 建筑轮廓、LoD1 建筑体块、基础高度
- 步行导航图、车辆导航图
- 建筑用途标签、入口候选点、安全区、疏散点、感染源点、封锁节点
- 平民、感染者、军队三类 NPC 及其行为参数
- 少量基础环境占位：车辆、路障、围栏
- 热力图、事件标记、轨迹回放、统计面板、参数配置

这套资产已经足以支撑一个“真实街区导入 -> 自动生成场景 -> 运行模拟 -> 回放分析”的原型。

## 6. 第一人称扩展所需新增资产

如果未来扩展到第一人称或近距离观察模式，至少还需要补充以下资产：

- 更精确的建筑立面
- 更可靠的入口、门窗和台阶
- 更高分辨率的贴图和材质
- 更密集的街具与碰撞体
- 室内模板、楼梯、电梯、房间布局
- 局部高保真地标或英雄街区

这部分不应进入第一版的强制范围。

## 7. 当前建议

第一版资产策略建议概括为：

- 优先做“世界骨架资产”和“语义与模拟资产”
- 场景几何保持 LoD1 到 LoD1.5 水平即可
- 角色和道具采用统一风格化低模方案
- 所有会影响模拟结果的资产优先于纯视觉资产
- 将第一人称所需资产明确隔离，避免其反向拖慢首发版本
