# 当前冻结基线

## 1. 文档目的

本文档用于冻结当前阶段的关键决策，避免主线在实现初期持续漂移。

冻结不代表永不修改，而表示：

- 在当前里程碑完成前，这些决策默认不再重新讨论
- 如需变更，应明确记录变更原因和影响范围

## 2. 冻结版本

- 冻结版本：`baseline-v1`
- 冻结日期：`2026-04-20`

## 3. 首发目标冻结

当前首发目标冻结为：

- 真实空间驱动的上帝视角群体模拟原型
- 不是第一人称沉浸式体验
- 不是开放世界写实城市游戏
- 不是多人联网游戏

## 4. 技术栈冻结

当前阶段技术栈冻结为：

- 离线数据与规则流水线：`Python`
- 资产编译：`Blender`
- 运行时模拟与可视化：`Godot`
- 数据交换：`JSON + glTF`

当前阶段不重新讨论：

- `Unity`
- `Unreal`
- 纯 `Web` 主运行时方案

## 5. 首发地图冻结

当前首发地图冻结为：

- 城市：`成都`
- 目标地点：`四川大学江安校区`
- 工程区域：`四川大学江安校区核心区`

当前冻结说明：

- “四川大学江安校区”是首发地图的现实语义锚点
- 第一阶段不会尝试覆盖整个校园的全部边界
- 第一阶段采用校园核心区近似 bbox，优先服务原型闭环

当前工程配置文件：

- [chengdu_scu_jiangan_core.json](/home/void0312/Workshop/urban-sim-lab/configs/areas/chengdu_scu_jiangan_core.json)

冻结依据：

- 四川大学官方公开地址指向江安校区位于成都市双流区川大路
- `川大江安校区站` 可作为校园核心区的近邻定位参考点

说明：

- 当前 bbox 是工程推断范围，不是官方校园红线边界

## 6. 首发范围冻结

第一阶段默认只做：

- 单一真实区域
- 单张地图
- 三类主体：平民、感染者、军队
- 俯视角或 2.5D
- 宏观可观察、可控制、可回放的模拟

## 7. 首发不做项冻结

在当前里程碑完成前，以下内容冻结为“不做”：

- 第一人称完整交互
- 室内真实重建
- 高保真立面还原
- 多人联网
- 超大地图
- 写实角色管线
- 复杂枪械手感系统
- 商业化部署与发布流程

## 8. 数据策略冻结

当前阶段数据策略冻结为：

- 优先使用 `OSM`
- 优先建立稳定的规范化层和运行时包
- 先做规则推导，不优先做复杂 ML 补全

当前阶段不要求：

- 多源融合必须完整
- 立刻接入街景视觉模型
- 立刻接入 DEM/LiDAR

## 9. 交付目标冻结

当前里程碑的真实交付目标冻结为：

1. 一个真实区域可被导入为规范化城市包
2. 规范化城市包可被构建为 runtime pack
3. Godot 可加载 runtime pack 并显示核心场景
4. 三类实体可在图上基础移动
5. 一次最小模拟可运行并输出结果

## 10. 解冻条件

只有在以下条件满足后，才建议重新讨论冻结项：

- Godot 场景加载闭环完成
- 第一局可运行模拟闭环完成
- 已有一次对真实区域的人工真实性校核

在这之前，默认遵守本冻结基线。
