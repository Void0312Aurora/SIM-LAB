# 国内数据源与本地研究模式

## 1. 文档目的

本文档用于补充中国场景下更适合本项目的地图、影像、POI、路径与官方校核来源，并定义“本地研究模式”的使用边界与落盘方式。

它服务于两个目标：

- 在中国区域中，补足单纯依赖 OSM 时常见的名称、步行网络、出入口和校园内部语义缺失
- 在不做二次分发的前提下，为本地研究和原型验证预留更宽松但仍然可控的工程空间

## 2. 当前假设

当前阶段默认假设如下：

- 当前项目是个人或小团队内部研究原型
- 当前不做商业化发布
- 当前不做二次分发、公开打包共享或出售原始第三方数据
- 当前仍然需要尊重各开放平台和官方服务的使用条款

这意味着：

- 我们可以更积极地把国内开放服务接入为“研究增强层”
- 但不应把受限制平台的原始 API 数据直接沉淀为可再分发的主资产库

## 3. 工程策略

建议将国内数据源按职责拆成三层：

### 3.1 可沉淀主资产层

建议用于长期沉淀为项目 canonical 数据：

- `OpenStreetMap`
- 团队人工整理后的校园边界、功能分区、出入口、命名锚点
- 学校或政府公开图件整理后的派生结果

这些数据应进入：

- `data/normalized/`
- `data/runtime/`
- 后续 `configs/overlays/` 或类似人工标注目录

### 3.2 在线增强与校核层

建议用于研究、比对、补全和人工核验：

- `腾讯位置服务`
- `百度地图开放平台`
- `高德开放平台`
- `天地图` 地名搜索、影像、矢量注记

这些来源更适合：

- 补充 `POI` 名称
- 补充 `步行路径` 语义
- 补充 `园区子点/出入口`
- 对照校核建筑、道路、校门和校园内部空间

### 3.3 官方锚点层

建议作为几何范围和命名可信锚点：

- 四川大学官方江安校区地图
- 地方自然资源和规划部门公开服务入口
- 自然资源部标准地图与天地图体系

这类来源适合用于：

- 校园边界裁剪
- 建筑命名校核
- 图层对齐时的真实性判断

## 4. 推荐国内来源

### 4.1 天地图

官方入口：

- 地图服务：<https://lbs.tianditu.gov.cn/server/MapService.html>
- 地名搜索：<https://lbs.tianditu.gov.cn/server/search.html>

可提供：

- 矢量底图
- 影像底图
- 注记图层
- 地名搜索结果

适合用途：

- 作为中国区默认底图和影像参照
- 在预览页中增加底图对照层
- 对照校核道路、地块、水体和地名分布

建议接入方式：

- 默认作为在线参考底图，不直接纳入主资产仓库
- 如需保留研究结果，仅保留本地截图、人工记录和派生标注，不直接分发瓦片

### 4.2 腾讯位置服务

官方入口：

- 地点搜索：<https://lbs.qq.com/service/webService/webServiceGuide/webServiceSearch>
- 路线规划：<https://lbs.qq.com/service/webService/webServiceGuide/webServiceRoute>

文档中明确可提供：

- 地点搜索结果
- 地址、分类、坐标
- `subpois`
- 驾车、步行、骑行、公交等路线规划

适合用途：

- 增强 `POI` 与地名
- 校核大型园区内部的子点、出入口和目的地引导
- 辅助生成或校核 `walk` 网络

建议接入方式：

- 作为在线增强层
- 将查询结果整理为人工核验记录或派生 overlay
- 不将原始 API 数据作为项目永久 canonical 主库

### 4.3 百度地图开放平台

官方入口：

- 地理数据服务：<https://lbsyun.baidu.com/data>
- 使用条款：<https://lbsyun.baidu.com/law.htm>

官方说明中可见：

- 可通过 API 创建地图应用，使用搜索等相关功能
- 平台支持管理和发布自有地理数据
- 使用条款明确限制对内部数据的直接存取和未授权抓取

适合用途：

- 名称、地址、POI 和对照校核
- 研究阶段的地图展示与人工比对

建议接入方式：

- 作为在线增强层与人工核验参考
- 如需本地保存，仅保留研究记录、人工摘要、派生标注和手工整理结果

### 4.4 高德开放平台

官方入口：

- 开放平台主页：<https://lbs.amap.com/>

说明：

- 高德是中国区重要候选来源，通常在道路、POI、出入口和导航体验上较强
- 当前自动抓取环境受该站点 `robots.txt` 限制，本文档暂未纳入已核验的具体 API 细节

当前建议：

- 将高德列为待人工核验后接入的候选增强源
- 在人工确认开放文档与使用条款后，再决定是否加入自动化研究流程

### 4.5 自然资源部标准地图服务

官方入口：

- 标准地图服务新闻说明：<https://www.mnr.gov.cn/dt/ywbb/201908/t20190802_2451215.html>

适合用途：

- 公开展示时的合规参考底图
- 中国区地图表达校核
- 项目文档、报告和演示背景图参考

说明：

- 标准地图更适合作为表达和校核参考，不适合作为校园级模拟的高细节几何主源

### 4.6 四川大学官方江安校区地图

官方入口：

- 江安校区地图：<https://dzb.scu.edu.cn/info/1025/1012.htm>

适合用途：

- 校园范围裁剪
- 建筑与功能区命名核对
- 校园内部空间真实性校核

当前意义：

- 这是当前项目在“江安校区”场景中的第一优先锚点来源之一

### 4.7 四川省自然资源厅数据服务入口

官方入口：

- 数据服务入口：<https://dnr.sc.gov.cn/scdnr/sjfw/sc_sjfw.shtml>

适合用途：

- 查询四川省和成都市自然资源系统公开入口
- 寻找地方时空平台、专题图件和规划公开资料

当前意义：

- 作为后续补充地方官方图件与服务入口的导航页

## 5. 推荐分工

针对成都四川大学江安校区，当前推荐组合如下：

- 基础几何：`OSM`
- 影像与底图参考：`天地图`
- POI、子点、出入口、步行路径增强：`腾讯位置服务`
- 名称与地址交叉校核：`百度地图开放平台`
- 校园边界与命名锚点：`四川大学官方江安校区地图`

在此基础上：

- `OSM` 继续作为主数据骨架
- 国内来源主要用来做校核、增强和人工整理
- 最终沉淀进项目仓库的是我们自己的规范化结果，而不是第三方原始服务结果

## 6. 本地研究模式

本地研究模式下，允许做的事情：

- 下载允许本地保存的公开图件和数据包供内部核验
- 保存研究截图、人工标注、核验记录和派生 overlay
- 在本地缓存 API 查询结果，供短期整理和比对

本地研究模式下，仍不建议做的事情：

- 将受限制平台的原始响应作为公开仓库内容提交
- 将第三方原始瓦片、批量 POI 或内部格式数据打包分发
- 用未经确认条款允许的方式进行大规模抓取

## 7. 本地落盘建议

建议使用如下目录约定：

```text
data/research/
  tdt/
    raw/
    notes/
    derived/
  tencent/
    raw/
    notes/
    derived/
  baidu/
    raw/
    notes/
    derived/
  scu/
    raw/
    notes/
    derived/
```

其中：

- `raw/`：原始下载包、截图、临时查询结果
- `notes/`：人工核验记录、来源说明、对齐备注
- `derived/`：经人工整理后形成的本地研究中间件

建议每一批研究数据至少记录：

- 来源平台
- 官方入口 URL
- 获取日期
- 查询范围
- 查询关键词或接口参数
- 是否允许长期本地保存
- 是否允许导出到项目 canonical 层

## 8. 当前建议的接入顺序

### 阶段 1：范围裁剪

- 根据四川大学官方江安校区地图整理 `campus polygon`
- 在预览中增加“校内范围”开关
- 把学校外部空间从主视图中弱化或裁掉

### 阶段 2：底图对照

- 给预览页增加天地图底图或影像参考层
- 用于肉眼判断当前 `drive/walk/building` 是否对齐真实空间

### 阶段 3：名称与出入口增强

- 接入腾讯位置服务地点搜索
- 拉取校园相关 `POI`、`subpois`、出入口和步行路径线索
- 形成本地研究笔记和人工 overlay

### 阶段 4：人工整理入库

- 将核验后的校园边界、校门、主要楼宇名、功能区和关键步行轴线沉淀为项目自有 overlay
- 再由 overlay 驱动 `normalized city` 与 `runtime pack`

## 9. 当前结论

如果项目长期聚焦中国城市或校园区域，那么只依赖 `OSM` 不够。

当前最合理的工程路线不是直接替换 OSM，而是：

- 用 `OSM` 保证开放、可沉淀和可编辑
- 用国内平台补名字、步行语义、出入口和真实性
- 用官方校区地图和官方公共服务做最终锚点

这样既能保住工程可控性，也能明显提升中国场景下的数据可信度。

## 10. 第一批建议下载项

当前建议优先下载到 `data/research/` 的本地研究材料如下：

### 10.1 `scu/`

- 四川大学官方江安校区地图页面截图或可下载图件
- 人工框出的校区边界、多边形和主要校门位置
- 校园主要建筑、学院楼、实验楼、体育馆、医院等命名锚点记录

### 10.2 `tdt/`

- 江安校区核心区的天地图矢量底图参考截图
- 江安校区核心区的天地图影像底图参考截图
- 与当前 `drive/walk/buildings` 预览图的对照笔记

### 10.3 `tencent/`

- 关键词 `四川大学江安校区`、`江安校区 校门`、`江安校区 教学楼`、`江安校区 食堂` 的地点搜索结果
- 返回的 `subpois`、出入口或子点信息
- 关键锚点间的步行路线结果

### 10.4 `baidu/`

- 与腾讯相同关键词集合下的地点搜索结果
- 建筑名称、地址和分类的交叉校核记录
- 与官方校区图不一致的点位列表

### 10.5 后续导入目标

以上研究材料整理后，优先沉淀为项目自有 overlay：

- `campus_polygon`
- `campus_gates`
- `building_name_overrides`
- `walk_spine_lines`
- `poi_name_overrides`

这些 overlay 一旦稳定，就应进入项目自有的规范化或派生流程，而不是持续依赖人工查图。
