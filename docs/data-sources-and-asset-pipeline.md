# 公开数据与自动化资产生成调研

## 1. 文档目的

本文档用于整理当前项目可利用的公开数据源、可直接获取的数据类型、可进一步推导的资产，以及适合引入机器学习补全的环节。

目标不是“自动生成 AAA 级写实城市”，而是构建一个足够可信、可扩展、适合模拟和策略玩法的城市资产流水线。

当前产品假设：

- 首个版本以第三人称上帝视角或俯视角为主
- 首个版本更重视模拟可信度，而不是近距离写实表现
- 第一人称沉浸式体验可作为后续扩展方向，而不是首发要求

## 2. 总体判断

如果项目目标是街区级或城区级的模拟体验，那么“近乎全自动化”的资产生成是可行的。

最合适的路线不是让生成式模型从零创造整个城市，而是：

- 用公开 GIS 数据提供真实世界骨架
- 用规则系统推导导航、功能分区和模拟语义
- 用机器学习补足缺失字段和局部细节
- 用程序化方式生成最终可用的游戏资产

## 3. 资产分层

从工程角度看，项目中的“资产”不只是模型和贴图，还包括支撑模拟的结构化数据。

建议将资产分为以下几层：

- 事实层资产：道路、建筑轮廓、水体、绿地、POI、地址、地形
- 推导层资产：导航图、逃生图、封锁图、人口热图、危险热图
- 几何层资产：建筑体块、道路网格、地表材质、围栏、树木、街具
- 语义层资产：建筑用途、入口位置、可藏身点、可封锁点、军队控制节点
- 模拟层资产：NPC 出生点、居住点、工作点、巡逻点、感染扩散参数
- 表现层资产：立面材质、招牌、道具变体、局部高保真区域

## 4. 主要公开数据源

### 4.1 OpenStreetMap / Overture Maps

适合作为底层世界事实数据的核心来源。

可直接获得：

- 道路、路口、人行道、铁路、水体
- 建筑轮廓、部分建筑高度、楼层、屋顶信息
- POI、地址、土地利用、行政区
- 一部分入口、设施、公共服务点

优点：

- 数据结构丰富
- 社区维护活跃
- Overture 在工程使用上更统一，适合作为标准化输入

局限：

- 区域差异很大
- 建筑高度、入口、室内信息常常缺失
- 城市家具覆盖不稳定

适合用途：

- 构建道路图和建筑图
- 生成地块、功能区和导航骨架
- 作为其他数据源对齐的基底

### 4.2 Google Open Buildings

适合作为建筑 footprint 的补充数据源。

可直接获得：

- 大量建筑轮廓
- 建筑检测置信度

优点：

- 在部分区域可补足 OSM 中建筑缺失的问题

局限：

- 主要提供建筑几何，不提供完整语义
- 覆盖范围不是全球一致
- 官方国家列表不包含中国大陆

适合用途：

- 补建筑底图
- 做建筑存在性校验

### 4.3 Copernicus / Sentinel / ESA WorldCover

适合作为卫星影像和地表分类来源。

可直接获得：

- 卫星影像
- 土地覆盖分类
- 水体、植被、裸地、农田等地表语义

优点：

- 全球尺度可用
- 很适合程序化生成地表材质与环境分区

局限：

- 分辨率有限
- 对街道级细节帮助有限

适合用途：

- 地表材质映射
- 绿化、水体、开阔地识别
- 环境类型分区

### 4.4 DEM / LiDAR / OpenTopography / USGS 3DEP

适合作为高程和地形数据来源。

可直接获得：

- DEM 高程
- 部分区域 LiDAR 点云
- 局部高精度地表或建筑高度参考

优点：

- 能显著提高地形真实性
- 可用于校正建筑高度和视线遮挡

局限：

- 高质量数据覆盖高度依赖地区
- 全球一致性较差

适合用途：

- 地形生成
- 高差、坡度、可视域分析
- 遮挡与防线评估

### 4.5 Mapillary

适合作为街景图像和街道设施特征来源。

可直接获得：

- 街景图像
- 由计算机视觉提取的 map features
- 交通标志、路灯、消防栓、路侧设施等对象线索

优点：

- 对街具自动化帮助很大
- 可为立面和街道细节补全提供依据

局限：

- 覆盖强依赖用户拍摄密度
- 连续性和完整性不稳定

适合用途：

- 街具摆放
- 道路设施补全
- 立面风格参考

### 4.6 OpenAerialMap

适合作为公开许可的航空影像补充来源。

可直接获得：

- 部分地区高质量航空或无人机影像

适合用途：

- 局部区域校核
- 精细化纹理参考
- 小范围重点街区重建

### 4.7 OpenAddresses / National Address Database / WorldPop

适合作为地址和人口热度来源。

可直接获得：

- 地址点
- 建筑地址关联
- 栅格化人口分布

适合用途：

- 推断居住密度
- 生成人口热图
- 生成平民 NPC 的居住和聚集权重

### 4.8 OSM 室内数据

可作为补充来源，但不适合作为主依赖。

可直接获得：

- 少量建筑的楼层、房间、通道、入口

局限：

- 覆盖稀疏
- 标注不统一

适合用途：

- 少量样板建筑参考
- 用于程序化室内模板的规则设计，而不是直接重建整城室内

## 5. 从公开数据可直接得到的资产

以下资产通常不需要复杂 ML，即可通过数据清洗或规则转换生成：

- 道路网络
- 建筑 footprint
- 建筑体块的底面和基础高度
- 绿地、水体、空地分区
- 土地利用图
- POI 图层
- 行政边界或街区切片
- 基础导航面
- 基础地形

这些内容足以支撑第一版俯视角模拟原型。

## 6. 可由规则进一步推导的资产

这是最关键的一层，也是最适合做“自动化”的部分。

从道路、建筑和地块数据可推导：

- 车辆通行图
- 步行通行图
- 疏散路径图
- 封锁节点图
- 瓶颈点和拥堵风险点
- 军队设卡点
- 平民逃生目标点
- 感染传播捷径和高密度传播区

从建筑数据可推导：

- LoD1/LoD1.5 建筑几何
- 建筑容量估计
- 建筑用途概率
- 可进入性评分
- 屋顶可达性近似
- 遮挡体与掩体价值

从人口、POI、土地覆盖可推导：

- 昼夜人口热图
- 商业活跃区
- 居住集中区
- 医疗与安全设施重点区
- NPC 初始生成权重
- 事件触发热点

## 7. 适合引入机器学习补全的部分

ML 最适合补全“缺失字段”和“中层细节”，不适合担任地图事实来源。

### 7.1 航拍或卫星影像补全

可做：

- 建筑缺失检测
- 停车场、围墙、开放空间识别
- 屋顶类型分类
- 绿化与障碍物分割

### 7.2 街景图像补全

可做：

- 立面材质分类
- 门窗和入口位置概率预测
- 招牌、雨棚、广告牌等立面元素识别
- 路灯、栏杆、消防栓、垃圾桶等街具定位

### 7.3 单目深度和分割

适合工具方向：

- `SAM 2` 用于图像分割
- `Depth Anything` 用于深度估计

适合用途：

- 从街景中推导近似空间层次
- 补充立面结构和街具布局

### 7.4 局部高保真重建

如果未来要对重点街区或地标做高保真表现，可考虑：

- 摄影测量
- `OpenDroneMap`
- `3D Gaussian Splatting`

这条路线更适合“英雄区域”，不适合作为整城默认方案。

## 8. 推荐的自动化资产流水线

### 阶段 1：事实层采集

导入并统一以下数据：

- OSM 或 Overture
- 建筑补充数据
- DEM 或 LiDAR
- 土地覆盖数据
- 人口和地址数据
- 街景和街景特征

### 阶段 2：规范化与清洗

构建内部统一 schema，解决：

- 坐标系统一
- 重叠建筑处理
- 悬空道路或断裂路网修正
- 建筑和地块关联
- POI 与建筑关联

### 阶段 3：语义推导

自动生成：

- 建筑用途标签
- 入口候选点
- 道路等级
- 可步行区
- 避难点和热点
- 封锁点和军队控制点

### 阶段 4：几何生成

程序化生成：

- 建筑体块模型
- 道路和路口网格
- 地表材质
- 绿化和围栏
- 街具占位模型

### 阶段 5：模拟资产生成

生成模拟需要的结构化资产：

- 平民居住点
- 工作点和目标点
- 感染扩散源点
- 军队巡逻或封锁节点
- 事件脚本触发点

### 阶段 6：人工审计与局部修饰

即使总体路线追求自动化，也建议保留少量人工校正能力，用于：

- 修复错误拓扑
- 修正关键建筑用途
- 指定地标建筑
- 增强演示区域观感

## 9. 精度预期建议

如果首个版本以第三人称上帝视角为主，建议精度目标如下：

- 道路和建筑轮廓：较高准确度
- 建筑高度：近似可信即可
- 立面细节：只需风格一致，不必逐栋还原
- 室内结构：可暂时不做真实重建
- 街具：关键区域有代表性即可，不追求全覆盖
- 纹理：优先表达区域气质，而不是逐物体写实

这套精度对于策略、模拟和宏观观察已经足够。

如果未来扩展到第一人称，则需要额外提升：

- 立面开口准确度
- 门窗位置
- 材质分辨率
- 人行设施密度
- 室内入口与过渡空间
- 小尺度碰撞与遮挡质量

## 10. 当前建议

现阶段最值得坚持的原则是：

- 先做俯视角/上帝视角
- 先做街区级自动生成
- 先做“结构可信”而不是“近看精美”
- 先把模拟资产做对，再补表现资产

只要底层数据与生成链条是对的，后续无论加第一人称、做高保真街区，还是扩展到更大范围，都会顺得多。
